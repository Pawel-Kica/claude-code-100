REST vs GraphQL vs gRPC

One internal service talking to a web client and three other services. Which API style?

DimensionRESTGraphQLgRPC
Best fitCRUD, public APIsRich clients, many shapesService-to-service
Payload shapeJSON, often over-fetchesNo over-fetchingBinary protobuf, smallest on the wire
LatencyFineFineLowest (binary protobuf)
Browser supportNativeNativeNeeds a proxy
Schema / typesOpenAPI, optionalBuilt-inBuilt-in (.proto)
CachingHTTP caching for freeDIYDIY
Learning curveLowMediumMedium-high

Verdict

Two styles, not one. Use REST for the web client: it's native, cacheable, and nobody needs a tutorial. Use gRPC for the service-to-service hops where latency and a shared .proto contract earn their keep. Reach for GraphQL only if the client genuinely needs many different shapes from one round trip; otherwise it's a server you now have to babysit.
